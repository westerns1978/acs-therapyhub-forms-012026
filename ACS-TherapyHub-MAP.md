## 3. Architecture principles (non-negotiable â€” apply to all work)

1. **Narrate-only.** A deterministic engine decides every count/verdict/level; a credentialed human confirms or overrides (logged with reason); AI writes **prose only**, never a number or a clinical/legal verdict.
2. **Witness, don't assume.** Prove behavior â€” anon RLS probes â†’ 0, forged-claims rejected, replay throwing 23505, a balance landing at $0, a truth table printed, eyes on the live screen. "It deploys" â‰  "it works."
3. **Present-then-apply** for any live-DB migration. Show SQL â†’ human approves â†’ apply â†’ verify between steps. Never auto-apply.
4. **Source-of-truth in git.** Commit the `.sql` even for MCP-applied migrations (they replay 1â†’N).
5. **Verify RLS through the authenticated app, never `execute_sql`** (MCP runs as service_role â†’ bypasses RLS â†’ false pass).
6. **New tables ship with scoped RLS**, never `Allow all`. Balance always **derives from the ledger**. The **$249 supplemental is a state pass-through, never revenue.**
7. **Merge â†’ build â†’ deploy**, in that order (stale-bundle bug otherwise). Deploy to `hosting:acs-therapyhub`.
8. **One-person process.** Branch â†’ verify â†’ merge â†’ deploy. No PR-review ceremony.
9. **Gates must agree.** A UI role gate must match its RLS predicate exactly (a Client must never see a staff surface).
